## Key structural finding (the portability hinge)

OpenSpace's `skill_applied` and `task_completed` are **LLM judgments over the execution transcript**, not raw execution telemetry. `SkillJudgment.skill_applied` (types.py:171-179) is set by the analysis LLM (`ExecutionAnalysis.analyzed_by`, types.py:271) reading what the agent did â€” "WORKFLOW: agent followed the prescribed steps / TOOL_GUIDE: agent used the described tool / REFERENCE: knowledge influenced decisions." The counters are then incremented atomically in SQL (store.py:572-595). So OpenSpace's *skill-level* effectiveness rides on a transcript reader â€” which skill-concierge already has (skill-usage-audit reads `~/.claude/projects/**/*.jsonl`, SKILL.md:27). Only the *tool-level* quality subsystem (quality/types.py) is genuinely execution-bound. **That split is the whole answer.**

## Bottom line

The single highest-leverage upgrade is the **follow-through judge** â€” adopt OpenSpace's `skill_applied` as an LLM transcript verdict (types.py:171-179, store.py:573), not a tool-fired counter. Today skill-concierge's best usefulness proxy, offerâ†’take (analyze.py:112-132), conflates "the skill was invoked" with "the skill helped"; splitting taken turns into *took-and-followed* vs *took-and-ignored* converts pure gate-compliance telemetry into a real usefulness signal, and it needs nothing sc doesn't already have â€” the transcript reader (skill-usage-audit) and the post-hoc analysis cadence both exist. Pair it with an **abandon/re-search rate** (OpenSpace's `fallback_rate`, store.py:579-583; computable from `search`-after-`auto` in existing ledger windows, analyze.py:238-240 â€” but rename to avoid colliding with analyze.py's infra `fallback`). What is genuinely impossible to port is **`completion_rate` / `task_completed`** and the entire execution-telemetry quality layer (`success_rate`, latency, failure streaks): those require running the task to a known outcome, which OpenSpace does in a harness and skill-concierge, as a selection-only layer, structurally cannot â€” so the honest metric stops at "offered â†’ followed," never claiming "â†’ succeeded." All four candidates sit **outside** the current usefulness-rate-upgrades plan (which covers body-trigger retrieval, AUTHORIZED-SKIP gating, find-skills escalation, audit patch â€” plan.md:27-32), so they build past it rather than duplicate it.
