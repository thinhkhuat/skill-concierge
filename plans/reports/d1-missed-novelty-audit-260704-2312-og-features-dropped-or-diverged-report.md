## Cross-cutting read
- **Engine integrity is high.** Every loud-failure guard (warning, dark/stale health, dim-mismatch, non-zero exit) survived the multivector + body-trigger patches untouched and is still test-covered. No BROKEN verdicts.
- **The losses are all at the "packaging + persuasion" layer**, not the retrieval core: (a) the portable service-free *default* became Docker-gated (the biggest real regression), (b) the cache rebuttal vanished, (c) the recall number isn't published even though the harness to produce it exists.
- **Two things got better**, not worse: token-proof is now measured on the fork's actual 507-skill index (17.8% window reclaimed), and the unit-test suite grew to cover the fork's patches.

## Unresolved / flags
- **Service-free recoverability (baseline open Q):** *architecturally recoverable* (engine supports embedded), but `setup.sh` has no `--service-free` path — it hard-exits on no-Docker. If portability matters, that's a ~10-line setup.sh fix, not a rewrite.
- **precision_eval as a real bar:** it depends on a shadow enrichment collection and authored scenarios; I did not run it (read-only, and it needs a live Qdrant). Whether it yields a trustworthy recall@k on the live `claude_skills` collection is UNVERIFIED here — D3's territory.
- **Note for the team:** the vendored `pyproject.toml:24-28` fastembed pin comment references "the live **mpnet** index" — i.e. the *vendored source is NOT pristine 0.1.0* even outside `server.py`. Minor, but it means "vendored = untouched upstream" isn't literally true; flag for whoever re-vendors.
